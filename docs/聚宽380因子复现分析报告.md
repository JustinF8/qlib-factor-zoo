### 2.1 basics 类（37个）— 存在部分问题

### 2.3 growth 类（9个）— 核心问题

| 因子名 | 问题 |
|--------|------|

## 四、推荐的Qlib复现路线图

### 第一阶段：技术指标因子（立即可做，~70个）

在现有 TDXGS handler.py 基础上扩展，新增以下因子组：

```
# 在 handler.py 的 TDXGS.get_feature_config() 中新增：

# 28. Elder 多空力道
fields.append("($high - EMA($close, 13))")  → bear_power
fields.append("(EMA($close, 13) - $low)")   → bull_power（取反即bear）

# 29. Aroon 指标
fields.append("(1 - IdxMax($high, 25)/25)")  → arron_down
fields.append("(1 - IdxMin($low, 25)/25)")   → arron_up

# 30. 量价趋势
fields.append("$volume * ($close-Ref($close,1))/(Ref($close,1)+1e-12)")  → VPT

# 31. 威廉变异离散量
fields.append("(($close-$open)/($high-$low+1e-12))*$volume")  → WVAD

# 32. 成交量震荡
fields.append("100*(EMA($volume,12)-EMA($volume,26))/EMA($volume,26)")  → VOSC

# 33. 量MACD (已在Alpha158中有VMA/VSTD等)

# 34. 52周位置
fields.append("($close-Min($close,252))/(Max($close,252)-Min($close,252)+1e-12)")

# 35. PLRC (线性回归斜率替代)
fields.append("Slope($close, 6)/$close")   → 替代PLRC6
fields.append("Slope($close, 12)/$close")  → 替代PLRC12
fields.append("Slope($close, 24)/$close")  → 替代PLRC24

# 36. KDJ 系列
fields.append("RSV($close, $high, $low, 9)")       → RSV
fields.append("SMA(RSV($close,$high,$low,9), 3, 1)") → K值
fields.append("SMA(K, 3, 1)")                       → D值
fields.append("3*K - 2*D")                          → J值

# 37. 成交量衍生
fields.append("$volume/Mean($volume, 5)")   → 量比5
fields.append("$volume/Mean($volume, 10)")  → 量比10
fields.append("$volume/Mean($volume, 20)")  → 量比20
```

### 第二阶段：风险/风格因子（需要截面计算，~46个）

risk(16) + style(30) = 46个因子可通过Qlib的表达式引擎在截面维度实现：
- `Kurtosis20/60/120` → 可用 `Kurtosis($close, N)` 表达式
- `Skewness20/60/120` → 可用 `Skew($close, N)` 表达式  
- `Variance20/60/120` → 可用 `Std($close, N)^2`
- `sharpe_ratio_*` → 可用 `Mean($close/Ref($close,1)-1, N)/Std($close/Ref($close,1)-1, N)`
- `beta` → 可用 `Slope` 或自定义算子
- 风格因子 → Barra模型，需要截面回归框架

### 第三阶段：财务因子（需要引入财务数据源，~194个）

需要将聚宽的季度财务数据导入Qlib数据存储，然后通过Qlib表达式引擎实现：
- basics(37) + growth(9) + pershare(15) + quality(71) + style_pro(16)

## 六、最终结论

1. **该脚本不可直接用于生产环境**：173个因子为空壳（54.5%未实现），已实现部分存在约15个字段名错误、8个公式错误、5个标量/Series混用问题。

2. **技术指标类因子（emotion + momentum 共75个）是脚本中质量最高的部分**，其中约60个可通过Qlib TDXGS现有算子直接复现，10个需要新增少量算子。

3. **Qlib已有算子（TDXGS + Alpha158 + custom_ops）可直接覆盖约60个技术指标因子**，无需额外开发。

4. **财务类因子（basics/growth/pershare/quality 共132个）** 需要引入聚宽财务数据源后才能复现，当前Qlib日频数据无法支持。

5. **风险/风格类因子（risk/style/style_pro 共62个）** 部分可在Qlib中通过内置算子实现（如Kurtosis/Skewness/Variance/Sharpe），部分需要Barra截面回归框架。
